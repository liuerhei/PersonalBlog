软硬件接口: 存储器层次结构

发表于 2021-09-25 更新于 2021-09-26 阅读次数： Valine：
本文字数： 2.2k 阅读时长 ≈ 2 分钟

  上一篇文章我们简单分析了流水线，其中取址和译码等多个阶段都涉及到计算机系统中一个重要的子系统：存储器系统。由于处理器的快速发展，导致存储器和处理器的时钟频率差距越来越大，使得存储器的性能成为制约计算机性能的重要瓶颈。例如，访存阶段需要很久才可以返回结果，那么整个流水线的效率就会很差。所以，可以利用程序的局部性原理设计层次化的存储系统来解决，如图1所示。在这种结构中，存储器的容量和访问时间随着离处理器距离的增加而增加。例如，最高层L0，是少量快速的CPU寄存器，CPU可以在一个时钟周期内访问访问它们。接下来是一个或多个小型到中型的基于SRAM的高速缓存存储器，CPU可以在几个时钟周期内完成访问。接下来是容量较大的基于DRAM的主存，可以在几十到几百的时钟周期内完成访问。接下来是慢速但容量较大的本地磁盘甚至是网络文件系统等。

图1:层次化储存结构

cache初探

  相信大家对cache都不陌生，它作为主存在更大的慢速设备中的数据缓冲区。在上文流水线小结中，我们就使用cache来替代存储器。cache对于编程人员是不可见的，由操作系统来维护管理。

  我们想象这样一个计算机系统，存储器按字节寻址，存储地址是m位，一共有M=2m个地址。高速缓存cache由S=2s个组构成，每个组中包含E个高速缓存行(cache line)。每个行包含一个存储B=2b字节的数据块，一个标示当前行是否有意义的有效位和一个t=m-(s+b)个标记位构成，如图2(a)所示。高速缓存的结构将地址划分为t个标记位、s个组索引位和b个块偏移位，如图2(b)所示。

图2:高速缓存的通用组织结构

  我们用元组(S,E,B,m)来描述高速缓存的结构。所以整个cache的大小是S*E*(B+m-(s+b)+1)。但实际上高速缓存的大小指的是所有块的大小之和，不包含标记位和有效位，所以cache大小C=S*E*B。根据高速缓存的组数和组中行数的不同，我们将高速缓存的映射策略分为：全相联、组相联和直接映射。其中全相联相当于高速缓存仅有一个包含多行的组(S=1)，组相联由多个包含多行的组构成，而直接映射相当于多个包含单行的组(E=1)构成。所以可以将全相联和直接映射看作是组相联的特殊情况，下面将对组相联进行分析。

对于有良好空间局部性的程序来说，如果使用高位地址作为组索引，那么相邻的数据块被映射到同一个组。如果组内的行数较少，会存在较多的数据块更换。导致高速缓存的使用率较低。

高速缓存：组相联

一个重要的前提：组中的任何一行都可以包含任何映射到这个组的数据块。

图3:组相联映射的组选择

  也就是说，对于一个给定的m位地址，我们先确定它在cache中的组号如图3所示。然后依次搜索组中的每一行，寻找标记和地址中的标记匹配的有效行。如果存在就命中高速缓存，从数据块中选择一个字节，如图4所示。如果不存在意味着未命中高速缓存，需要从存储器中取出包含该地址的块。一旦高速缓存获取数据块，就需要使用替换策略将组中另一个块换出，常用的策略是最近最少使用（Least-Recently-Used，LRU）策略。

图4:组相联的行匹配和字节选择

cache缺失

如果未命中高速缓存，即cache缺失，需要从存储器中去数据并更新cache。这一小节讨论下cache缺失的处理过程。这里需要额外的一些工作，需要两部分共同完成：处理器控制单元，主存访问和更新cache的独立控制器。cache缺失触发数据冒险，需要阻塞流水线等主存操作完成。对于乱序执行的处理来说，依然可以执行其他一些指令，这里不做分析。

这里我们来分析下指令缺失时的处理过程，数据缺失的情况要更简单一些。如果取指时发生缺失，即指令缓存中的内容无效，需要从主存中读取数据。由于执行的第一个时钟周期已经完成PC的更新，因此缺失的指令地址等于PC-4。地址产生后通过独立控制器从主存获取指令，然后使用返回结果更新cache。整体步骤如下：

  1. 程序计数器PC的值减4。
  2. 主存的读操作，并等待结果返回。
  3. 更新cache，包括替换数据块，更新标记位和设置有效位。
  4. 重启指令执行的步骤，从cache中读取指令内容。

数据访问缺失时和指令缺失类似，需要独立控制器根据地址从主存中读取并返回数据，同时更新cache。

cache写

  写操作和读操作略有不同。假设一个store指令，我们只将数据写入数据cache，这会造成主存和cache的不一致问题。解决这个问题最简单的方案是写直达，即写操作总是同时更新cache和主存。如果当前字不在cache中（写缺失），我们需要先从主存中取出数据并写入cache，然后将引起cache缺失的数据重新写入cache，同时更新主存中的数据。
